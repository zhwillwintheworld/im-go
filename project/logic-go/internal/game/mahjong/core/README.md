# 定义
这个目录是为了 定义麻将游戏的核心流程，但是不解决差异，主要负责负责核心流程的服务调用，定义流程的公共字段以及接口，让各个麻将的实现者，只需要关注差异即可

# 核心流程
1. 游戏开始，创建牌局
2. 动作
3. 任务判断
4. 分发任务
5. 任务执行
6. 状态分发
7. 结算


# 描述
我想设计一个 go 语言的麻将应用，我现在需要设计游戏本身的环节，例如 1.我需要抽象一个麻将游戏公共的模块（core），抽象出主流程例如各个麻将类型都有的流程环节，给出公共字段以及接口定义 2. 会同麻将 （湖南省会同县），需要定义生成牌局（牌只有 108张 只支持 万 条 筒），需要定义操作集合 （可以摸牌 出牌 碰 杠 胡，报听，抢杠 ） 还需要有结算配置（小胡 大胡 大大胡 烧庄 需要计算底分 ） 与用户自定义状态（支持报听 ），还有自定义胡牌算法 （例如 清一色，七小对，龙七对，将将胡，缺一门，报听胡（第一轮就可以胡 才可以报听），）还有胡牌类型 （胡，自摸，报听胡， 抢杠胡）  3. 太湖麻将 （安徽省太湖县）定义生成牌局 （牌有 136 张，支持 万条筒，东南西北中发白）
，定义操作集合（摸牌，出牌，碰，吃，杠，胡，花），结算配置（按照牌型 +花 +胡的类型+ 杠的数量 计算总花数 然后根据总花数/10 * 底分）用户自定义状态（支持临时杠数增长，支持花数量） 胡牌类型（无花果自摸 清一色 七小对  龙七对  碰碰胡 大吊车）还有胡牌类型 (胡 自摸)
